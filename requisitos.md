Requisitos do Sistema – EcoFood

Atores
Usuário
Sistema de Notificação: responsável por enviar alertas de vencimento

Requisitos Funcionais
RF01: O sistema deve permitir o cadastro de usuários
RF02: O sistema deve permitir que o usuário realize login
RF03: O sistema deve permitir o cadastro de alimentos
RF04: O sistema deve permitir editar e remover alimentos
RF05: O sistema deve permitir registrar a data de validade dos alimentos
RF06: O sistema deve emitir alertas quando um alimento estiver próximo do vencimento
RF07: O sistema deve exibir a lista de alimentos cadastrados
RF08: O sistema deve sugerir receitas com base nos alimentos disponíveis
RF09: O sistema deve permitir marcar alimentos como consumidos ou descartados
RF10: O sistema deve gerar relatórios de desperdício

Requisitos Não Funcionais
RNF01: O sistema deve possuir interface simples e intuitiva
RNF02: O tempo de resposta deve ser inferior a 2 segundos
RNF03: O sistema deve ser acessível via navegador (web)
RNF04: O sistema deve garantir a segurança dos dados dos usuários
RNF05: O sistema deve estar disponível 95% do tempo

Histórias de Usuário
Como usuário, quero cadastrar alimentos para controlar minha despensa
Como usuário, quero receber alertas para evitar desperdício
Como usuário, quero visualizar os alimentos disponíveis
Como usuário, quero sugestões de receitas para aproveitar melhor os alimentos
Como usuário, quero acompanhar meu desperdício para melhorar meus hábitos
